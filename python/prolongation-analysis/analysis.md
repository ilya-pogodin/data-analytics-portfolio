# Анализ пролонгации проектов (2022–2024)

**Задача:** рассчитать коэффициенты пролонгации K1/K2 по каждому аккаунт-менеджеру и отделу помесячно на основе финансовых данных о отгрузках.

**Стек:** Python, pandas, numpy. **Результат:** автоматический многостраничный Excel-отчёт.

> Исходные данные не публикуются из соображений конфиденциальности. В репозитории — методология, код и обезличенный итоговый отчёт (`report_anonymized.xlsx`).

## Этапы анализа

1. **Загрузка и подготовка данных** — приведение месяцев к единому формату, `melt` широкого формата, очистка сумм от пробелов и запятых, удаление служебных статусов («стоп», «в ноль», «end»).
2. **Агрегация отгрузок** по `id`, месяцу, году и аккаунту.
3. **Определение последнего месяца реализации** проектов (`groupby('id')['date'].max()`).
4. **Расчёт K1 и K2** по каждому месяцу 2023 года через цикл по `pd.date_range`:
   - **K1** = отгрузки в текущем месяце по проектам, завершённым месяц назад / сумма закрытых в прошлом месяце
   - **K2** = отгрузки по проектам, закрытым 2 месяца назад и не продлённым сразу / сумма таких проектов
5. **Сводные таблицы** (`pivot_table`) по месяцам и менеджерам, средние по отделу, годовой рейтинг менеджеров.
6. **Экспорт** многостраничного Excel-отчёта (8 листов: сырые коэффициенты, матрицы K1/K2, динамика отдела, годовые итоги, данные для графиков).
